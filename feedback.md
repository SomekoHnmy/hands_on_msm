## ① ロジカルエラー・コードの正しさに関わる問題

### 1. 安定化重みの分子と最終MSMの整合が取れていない（理論上の誤り）
- `04_msm_treatment_model.R:27-29` の分子モデルには baseline 共変量（`age_z`, `smoking`, `hypertension`, `dyslipidemia`）が入っていますが、最終MSM `07_msm_point_estimate.R:25` は `cvd_event ~ metformin_high + cumA + factor(month)` で baseline 共変量を含みません。分子に V（baseline共変量）を入れた安定化重みでは、疑似集団の中で治療は「Vで条件づけたときにのみ」ランダム化されたことになるため、最終MSMにもVを入れて（周辺効果が欲しければVで標準化して）推定するのが正しい手順です。今の組み合わせは V の交絡を打ち消しきれない形になっています。
- 実際に分子から V を外して再計算すると RD が `−0.223` → `−0.270` に動くことを確認しました（真値 `−0.232`）。無視できない差です。修正は「分子を `metformin_high_prev + factor(month)` のみにする」か「最終MSMにVを入れてV上で標準化する」のどちらか。`course_guide` §4-2 の「分子モデル：baseline と治療履歴だけで予測」という説明も、この要件（分子に入れた変数は最終モデルにも必要）に触れておくべきです。`10_advanced.R:101-110` も同じ構造です（3交絡版でMSMが `−0.254` と真値 `−0.277` からやや離れるのは、この影響も一因かもしれません）。

### 2. 自前クラスターロバストSEにIPW重みが掛かっていない（コードのバグ）
- `08_msm_confidence_interval.R:26` の `ef <- X * u` は、重み付きGLMのスコアに必要な prior weights が抜けています。`ef <- X * (u * weights(model))` に直すと `sandwich::vcovCL`（`cadjust=FALSE`）と小数4桁まで一致することを実行して確認しました。現状は簡易実装が sandwich より6〜7%小さいSEを返しており、48行目の「sandwich は小サンプル補正・自由度調整が入るため、わずかに値が異なります」という説明は誤った原因帰属です（差の主因は重みの掛け忘れ）。

### 3. 10_advanced.R の gfoRmula の参照介入が間違っている
- `10_advanced.R:206` で `ref_int = 0` としてコメントに「参照は『常に非増量』」とありますが、gfoRmula では `0` = `natural course`（自然経過） です。「常に非増量」は interventions リストの1番目なので `ref_int = 1` が正しい。`09_gformula.R:289` は `ref_int = 1` で正しく書けているので、10 だけずれています。このままだと出力のリスク差が自然経過との比較になり、直前のMSMの結果（増量 vs 非増量）と並べられず初学者は混乱します。

### 4. Cox の参考出力がメッセージと食い違う
- `02_prework_pooled_logistic_regression.R:82-84` は「HbA1c を入れるか入れないかで推定値が動く」と表示しますが、実行すると HR は `0.656` vs `0.660` でほぼ同じ。メッセージが出力と矛盾しており、初学者は「動いてないけど？」となります。数値が動く例として機能していないので、Cox 部分は削るかメッセージを直すべきです。

### 5. 真値・結果数値の版ずれ
- `data/true_values.csv` の真値は `−0.2319`（表示は `−0.232`）ですが、`course_guide`・`07:48` のハードコードは `−0.233`。事前課題を走らせると「真値 `−0.232`」と出て、ガイドの表（`−0.233`）と食い違います。
- MSM点推定も、実行すると `−0.223`（増量 `0.243` / 非増量 `0.467`）で、ガイドの `−0.225` と一致しません。おそらく古い版の数値がガイドに残っています。全数値を現行データで取り直して統一を。

### 6. 実行すると必ず出る警告への手当てがない
- `glm(..., family=binomial, weights=sw_trunc)` は非整数重みのため 「non-integer #successes in a binomial glm!」警告が必ず出ます（07・10で確認、他にも「50件以上の警告」と表示される）。初学者はここで確実に手が止まります。quasibinomial を使うか、「この警告は想定内で無害」というコメントを該当箇所に必ず入れてください。

### 7. 細かいが挙動に関わる点
- `01_prework_generate_person_data.R:78` のイベント割当（`m == n_months && ev$cvd_event == 1`）は「処方ログがイベント月でちょうど止まっている」というデータ生成側の暗黙の前提に依存しています。教材データでは正しく動きますが、「実データではイベント日と月区間を突き合わせる必要がある。ここは生成データの性質を利用したショートカット」という注記がないと、実データ解析の手本として誤学習させます。
- データ生成 `generate_data (not for participant use).R:129` で L=0 のとき `round(7.5 - abs(rnorm(1, 0.5, 0.3)), 1)` が約2%の確率で 7.5 に丸まり、`>= 7.5` の二値化で `hba1c_high=1` に誤分類されます（逆方向は起きない）。意図的でなければ軽微な測定誤差の混入です。
- 手書き g-formula のアウトカムモデル `09_gformula.R:40-42` には時間項がない一方、MSMは `factor(month)` 入り、同スクリプト後半の gfoRmula の `ymodel` は `t0` 入り。DGPのベースラインハザードが一定なので結果は正しいのですが、3つのモデルで扱いが不揃いで、初学者は「なぜここだけ月を入れないのか」と必ず引っかかります。
- `09_gformula.R:155-166` の bootstrap 内 `sim` は点推定で使った `mc = 50` のモンテカルロ複製をしていないため、CIにシミュレーション誤差が余分に乗ります（軽微、CIがやや広めに出る方向）。

## ② 初学者視点で分かりづらい説明

### 最優先：ガイドとファイル名の全面的な不一致
- `course_guide.md:25-31` は `01_prework.R` / `02_msm.R` / `03_gformula.R` / `04_advanced.R` を参照しますが、実ファイルは 01〜10 の別名です。本文中の参照（§4-3「`02_msm.R` は第1部…第2部…」など）もすべて旧構成前提。初学者が最初に開くドキュメントで、指示されたファイルが存在しないのは致命的なので、対応表を作るか全参照を書き換えてください。

### ガイドの記述とコードの矛盾
- `course_guide.md:118`「実務では cobalt・sandwich 等のパッケージを使いますが、教材では依存を減らすため base R で自前計算します」→ 実際のスクリプトは ggplot2・cobalt・sandwich を自動インストールして使用しています（03, 06, 08）。真逆の記述です。
- `course_guide` §3-1 は真値計算を事前課題の文脈で説明しますが、事前課題スクリプトは `true_values.csv` を読むだけで「← 当日に導出」と表示します。ところが当日のどのスクリプトが「導出」に当たるのか（実質は 09 の真係数シミュレーション）がどこにも書かれておらず、伏線が回収されたことに気づけません。

### 説明そのものの分かりづらさ
- **画像リンク切れ**：`prework_introduction.md:11` ほか `illustrations/scene_0〜2.jpeg` を参照しますが、リポジトリに illustrations フォルダが存在しません。
- **「breakthrough-stroke 論文」の書誌情報がない**：ガイド §4-5・§5-7・§7 とスクリプト補足で繰り返し比較対象にするのに、引用が一度も出てきません。参加者はどの論文の話か分からないまま「論文では〜」を聞かされます。
- **pooled logistic regression の位置づけが混乱を招く**：`prework_introduction.md:126` で「Lが治療と無関係に揺れるだけなら pooled logistic regression で何とかなる」と書く一方、事前課題では pooled logistic が「失敗する naive 手法」として登場し、当日のMSMの最終モデルも「重み付き pooled logistic」です。「ダメなのは pooled logistic ＋共変量調整であって、pooled logistic ＋重み付けは正しい道具」という整理をどこかで一度明示しないと、回帰経験者ほど混乱します。
- **risk_diff_from_glm の中身が説明されていない**：`02:40` では二値変数 `hba1c_high` に各月の観測平均（`0.55` のような連続値）を代入しています。何をしている計算なのか（そしてそれが naive な近似であること）のコメントがなく、コードを読める人ほど「二値に 0.55 を入れていいのか？」で止まります。
- **課題文の崩れた文**：`prework_introduction.md:142`「もし、どんな臨床家であっても薬剤を増量し、同じ量で維持したり減量することはありえない、という場合は…」は文として解読困難です。positivity の導入という大事な箇所なので書き直しを。
- `course_guide.md:51`「MSM も g-formula も必要になる唯一の状況の核心」――「唯一」は誤解を招きます（治療–交絡フィードバックは代表的な適用場面であって、唯一ではない）。
- **ESS の説明**（05:56-60）が「ESSが高い＝positivity良好」と断定調ですが、ESSは重みのばらつきの指標で positivity の間接的なチェックです。ガイド側は自覚的に書けているので、スクリプト側にも一行補足を。

### 軽微（誤字・残骸）
- 出力の誤字「調整 of 仕方」(`02:79`)、`exchangability` → `exchangeability`（`prework_introduction.md:100, 143`）、`marginal structure model` → `structural`（`prework_introduction.md:5`）。
- `agent/generate_data.R` は別DGP・別シード・他人のローカルパス（C:\Users\sangu\...）入りの残骸で、参加者が開くと確実に混乱します。削除推奨。
