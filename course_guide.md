## 0. なぜこのハンズオンをやるのか

MSM や g-formula は、時間依存交絡（time-varying confounding）を扱うための標準的な手法です。しかし、疫学を学習した人でも「名前は知っているが何をしているのか分からない」ことが多い。その一因は、既存の解説が「分かっている人が、分かっている人向けに」書かれていて、初学者が最初につまずく段差が放置されがちなことにあります。

このハンズオンでは、その段差を一つずつ埋めます。具体的には、初学者が必ず引っかかる次の5点に、教材の各所で手当てをしています。

1. **なぜ「調整」ではダメなのか**（共変量に入れるのと重み付けは何が違うのか）
2. **重みの分子・分母が何の確率なのか**
3. **g-formula の「各期間のハザードを掛け合わせる」の意味**（なぜ足さず掛けるのか）
4. **g-formula の「反実仮想の代入」が何をしているのか**（`A <- 1` の一行の意味）
5. **MSM と g-formula は同じestimand（推定量）を違うルートで推定していること**

### 💡 そのCQ、本当にMSMが要りますか？ ([00:21:22](https://www.youtube.com/watch?v=huS1yp6DtlI&t=1282s)〜)

MSM や g-formula は因果推論の万能装置ではなく、**時間依存性交絡を処理するための道具**にすぎません。「中間因子かつ交絡因子」という顔を持つ変数がいないのであれば、わざわざ使う必要はありません。

* **必要になる例**:
  * **ワルファリンと PT-INR**: INR（検査値）を見ながら用量を調整し、用量を変えると次の INR が動く。古典的な時間依存性交絡。
  * **パーキンソン病**: 体の動き（症状）を見ながら薬を増減し、薬が次の体の動きを変える。
* **必要にならない例**:
  * **脳卒中二次予防の抗血小板薬**: 凝固時間を定期的に見てアスピリンの量を調整する、ということは現実にはしない。虚血イベントが起きたら変更する、出血したら止める、という判断であって、一定間隔で変数を見て戦略を変えるわけではない。時点ごとの状態を織り込みたいだけなら `pooled logistic regression` で足ります。

自分のクリニカルクエスチョンにこの「時間依存性交絡」という役者がいるかどうかを先に考える。ここを飛ばして道具の話に入ってはいけない、というメッセージです。

## 2. 舞台設定：登場人物を具体物で固定する

しばしば抽象記号 `L`（時間依存性交絡因子）・`A`（治療）・`Y`（アウトカム） で話を進めると、どれがどれのことかわからなくなり、脱落しやすくなります。そのため**記号を必ず具体物とセットで**扱います。

* **A (治療)** = その月にメトホルミンの「高用量にある（=1, 初期用量 500mg より高い用量で治療中）／初期用量のまま（=0）」
* **L (時間依存性交絡因子)** = その月に HbA1c が「高値（=1）／良好（=0）」
* **Y (アウトカム)** = その月に心血管イベントが起きたか（起きた=1, 起きてない=0）
* **時間区間**: 1か月刻み × 6区間（6か月間フォロー）
* **Baseline共変量**（時間で変化しない初期共変量）: 年齢、性別、喫煙、高血圧、脂質異常症の5つ

### 2-1. 仮想世界の前提条件

題材は、**糖尿病患者に対するメトホルミンの用量調整と、心血管イベントの関係**です。

* 患者1,000人を6か月間フォローする。
* 毎月 HbA1c を測定し、それを見てメトホルミンを増量する/しないを決める。
* 年齢・性別・喫煙・高血圧・脂質異常症は観察期間中は変化しない（ベースライン固定）とする。
* 途中のデータベースからの脱落（打ち切り）や、死亡などの競合リスクは存在しないものとする。

動画の中で演者自身が繰り返し断っている通り、これは**かなり単純化された架空の世界**です。競合リスクや打ち切りの話を最初から始めると本題にたどりつけないので、あえて置いています。

ここで当日参加者から出た質問が重要なので、ぜひ聞いてみてください ([00:12:49](https://www.youtube.com/watch?v=huS1yp6DtlI&t=769s))。

> **質問**: 「今言っている仮定がもし成り立っていなかったら、この先の解析はどうなるんですか？」
>
> **回答**: 極端に言うと、仮定が成り立っていなければこの先どれほど複雑な解析をしても意味がなくなります。個別の仮定の変更への対応例は以下の通りです。

| 崩れる仮定                             | 現実の臨床シナリオ                     | 対処法                                                       |
| :------------------------------------- | :------------------------------------- | :----------------------------------------------------------- |
| **単剤治療である**               | メトホルミン以外の糖尿病薬も使う       | 治療戦略ごとにグループ（治療アーム）を分ける                 |
| **増量の定義が 500mg 超**        | 臨床的な増量の定義が異なる             | 閾値を変えればよいだけ                                       |
| **データベースからの脱落がない** | 途中で追跡不能になる（打ち切りがある） | 打ち切りに対する重みモデル（IPCW）を別途導入して掛け合わせる |
| **毎月の定期フォロー**           | 毎月ではなく3か月ごとのフォロー        | データセットを区切る期間（タイムグリッド）を変更する         |

このやりとりの結論が本質的です。
**「いろんな仮定を置かないと、そもそも当てはめられないモデルである。だからこそ事前にきっちりどのような仮定のもとにモデルを動かすのか明確にしておくことが大事」** ([00:14:21](https://www.youtube.com/watch?v=huS1yp6DtlI&t=861s)〜)。臨床感覚から見て仮定が明らかにおかしいなら、そもそもそのモデルの組み方が良くない、ということになります。

## 3. HbA1c の「三つの顔」と通常の回帰の限界

ここが今回のハンズオンの心臓部です。急がずに丁寧に理解してください。
今回のデータ構造（DAG：有向非巡回グラフ）において、HbA1c（L）は次の三つの動きを同時に行います ([00:17:18](https://www.youtube.com/watch?v=huS1yp6DtlI&t=1038s)〜)。

1. **A → L（治療の結果＝中間変数）**: メトホルミンを高用量にする → 次の月の HbA1c が下がる。
2. **L → A（治療の動機＝交絡因子）**: HbA1c が高い → 医師が「増量しよう」と判断しやすくなり、高用量になりやすくなる。
3. **L → Y（予後因子＝交絡因子）**: HbA1c が高い → 心血管イベントを起こしやすくなる。

この三つを重ねると、HbA1c は **中間因子（mediator）であり、同時に交絡因子（confounder）でもある** という二つの顔を持つことになります ([00:18:36](https://www.youtube.com/watch?v=huS1yp6DtlI&t=1116s)〜)。箱を横一列に並べた図では分かりにくいのですが、矢印を時間軸に沿って丁寧にたどると、この関係が見えてきます。

また、もう一つの重要な仮定として **「メトホルミンは HbA1c を介さなくても直接的に心血管疾患を抑制する効果がある」** ([00:15:46](https://www.youtube.com/watch?v=huS1yp6DtlI&t=946s)) を置いています。これがあるため、Lを経由しない直接効果の矢印が存在します。

> **【つまずき手当て 1：なぜ調整ではダメか】**
> ここが第一の関門です。「L（HbA1c）を共変量として回帰モデルに入れれば、交絡は調整できるのではないか？」という素朴な発想が、なぜ通用しないのかを説明します。
>
> * L を**調整に入れない** → ②③の交絡経路が開いたままになる → 交絡バイアスが生じる。
> * L を**調整に入れる** → ①の「増量 → HbA1c改善 → イベント減」という**本来評価したい因果効果の経路まで塞いでしまう**（過剰調整バイアス、中間変数バイアス）。
>
> つまり、「入れても入れなくても推定値が壊れる」ため、通常の回帰モデルによる調整というアプローチでは解決できません。そこで重み付け（MSM）やシミュレーション（g-formula）が必要になります。

## 4. MSMの3つの前提

MSM を用いるためには、以下の3つの基本的な仮定（前提条件）を満たしている必要があります。

1. **ポジティビティ (positivity)**: 各時点で、それぞれの治療群（増量群・維持群）に十分な患者が存在すること。「確率が正である（ゼロではない）」という意味です。重みの分布の重なりや有効サンプルサイズ（ESS）で間接的に診断します。
2. **交換可能性 (exchangeability)**: 比べる群同士が実質的に同じ（偏りがない）とみなせること。重み付け後の月次標準化平均差（SMD < 0.1）の確認につながります。
3. **一貫性 (consistency)**: 自分がCQで立てた治療介入と、データベースで実際に記録されている治療内容が一致していること。

これらは、MSM に限らず他の因果推論手法でも基本となる前提であり、特に target trial emulation（標的対照試験の模倣）をデザインする上できわめて重要です ([00:29:25](https://www.youtube.com/watch?v=huS1yp6DtlI&t=1765s))。

## 5. 事前課題（`01_prework_generate_person_data.R` & `02_prework_pooled_logistic_regression.R`）

事前課題の内容を簡単に説明します。もし関心があれば取り組んでみてください。

### 5-1. 真値：仕組みを知っているからこそ計算できる「正解」

シミュレーションデータの最大の強みは、データの発生ルールを自分で決めているため、**正しい答え（真値）を後から正確に計算できる**点にあります。
大規模な集団に対し、「全員を強制的に高用量にする介入」と「全員を強制的に初期用量のままにする介入」を行い、それぞれの6か月累積発生率の差を求めます。

* **真値：リスク差 = −0.232**
  * 常に高用量状態でいくと、6か月で心血管イベントを約23.2%ポイント減らす（常に増量した世界のリスク ≈ 0.25 / 常に非増量のリスク ≈ 0.49）。教材用にあえて効果を大きめに設定しています。

### 5-2. 生データを person-interval に畳み込む

事前課題では、実務に近いバラバラの生データ（4テーブル：患者マスタ `patients`、処方ログ `prescriptions`、検査ログ `labs`、イベントログ `events`）からスタートします。
これを、1人×月で1行の **person-interval 形式**に `for` ループで1ステップずつ畳み込みます。この過程で、「その月の用量が初期用量 500mg より高いか」で高用量状態（`metformin_high`）を導出し、HbA1c 実値を 7.5% で二値化します。本番の解析はすべてこの person-interval の上で動くため、データ構造に馴染んでおくことが重要です。参考のためにデータ定義書（`data_definition`）も同梱しています。

### 5-3. naive 解析：「調整してもしなくても真値に届かない」

通常の pooled logistic regression を用い、時間依存交絡 L を調整しないモデルと調整したモデルを実行します。

| 推定方法                | リスク差 (Risk Difference) |
| :---------------------- | :------------------------- |
| **真値**          | **−0.232**          |
| naive（L を調整しない） | −0.110                    |
| naive（L を調整する）   | −0.093                    |

* **結論**: L を入れても入れなくても、真値 −0.232 には届きません。調整の仕方で値は動きますが、どちらも正しくありません。通常の回帰モデルという枠組み自体が、この問題に対しては力不足であることを確認し、本番へ進みます。

## 6. 本番① MSM（`03` 〜 `08`）：治療割り付けをモデル化する

### 6-1. 考え方

「HbA1c が高い人ほど増量されやすい」という**治療選択の偏り（傾向）**をモデル化し、その逆確率で重み付けすることで偏りを打ち消します。これにより、あたかもランダムに治療が割り振られたかのような**疑似集団**を作り出し、その中で効果を比較します。

### 6-2. 安定化重み (SW) の2つのモデル ([00:34:57](https://www.youtube.com/watch?v=huS1yp6DtlI&t=2097s)〜)

* **分母モデル**: すべての交絡因子を入れて治療を予測します（時間依存交絡 L、前月の治療履歴 Aprev、前月のLprev、baseline共変量）。
* **分子モデル**: 重みを安定させるため、治療履歴（Aprev, Lprev）のみを予測因子として治療を予測します（分子には baseline 共変量を入れない、または入れた場合は最終モデルにも入れる必要があります）。
* **安定化重み (Stabilized Weight: SW)** = （分子モデルの予測確率）/（分母モデルの予測確率）を、個人ごとに時間方向に掛け合わせた累積積。

> **【つまずき手当て 2：重みの分子・分母が何の確率か】**
> スクリプト `05_msm_stabilized_weight.R` を実行すると、**具体的な1人の患者（ID=3）**のデータで重みの算出過程テーブルが出力されます。
> 「この人は HbA1c が高値（L=1）であるにもかかわらず、維持治療（A=0）を受けた。これは『受けにくい治療をあえて受けた人』であるため、分母の予測確率が小さくなり、逆数である重みが大きくなって疑似集団内で強くカウントされる」
> この実数値の動きを追うことで、確率の式が何を意味しているかを直感的に理解します。

### 6-4. 実務チェック（既習者向け見どころ）と数値の読み解き

実データで解析を行う際、定番パッケージを用いた以下のチェックが不可欠です。

1. **Positivity の確認（ESS）**:
   重み付けによって情報量がどれだけ残っているかを、各月の有効サンプルサイズ（ESS）で診断します。
   * *本データの数値*:
     * 月1: N=1000, ESS=947 (95%)
     * 月2: N=935, ESS=861 (92%)
     * 月3: N=873, ESS=789 (90%)
     * 月4: N=809, ESS=716 (89%)
     * 月5: N=763, ESS=659 (86%)
     * 月6: N=721, ESS=617 (86%)
   * 各月とも ESS が 86% 以上を維持しており、重みが極めて穏やかで positivity は極めて良好です。
   * *実務の注意点*: 実際のデータでは、高齢かつ高HbA1cの患者は全員増量されるなどして、月が進むにつれて特定の群が「しぼんで」消失し、ESSが激減して推定不可能になることがあります ([00:47:52](https://www.youtube.com/watch?v=huS1yp6DtlI&t=2872s)〜)。
2. **重み分布と Truncation**:
   * *本データの数値*: 切り詰め（Truncation）前で Min 0.262 / Median 0.865 / Mean 0.983 / Max 3.675。1%/99% で切り詰めると `[0.434, 2.195]` に収まります。
   * 演者いわく「この程度なら正直切り詰めなくても良いレベル」。しかし、実データでは重みが 1000 や 10000 に達することがあるため、実務ではお作法として Truncation が必要になる場面が多いと思われます。
3. **月次 SMD**:
   * *本データの数値*: 重み付け後も、一部の月で HbA1c の SMD が 0.2〜0.5 と、目安の 0.1 を超えてしまいます ([00:27:32](https://www.youtube.com/watch?v=huS1yp6DtlI&t=1652s)〜)。
   * 本データはあえて極端に作ってあるためバランスが完全には取れませんが、今回は手順の提示を目的としているため、取り繕わずにそのまま進めます。

## 7. 本番② parametric g-formula（`09_gformula.R`）：アウトカム側からモデル化する

### 7-1. 考え方

「もし全員が毎区間かならず高用量でいたら／初期用量のままだったら、イベントはどれくらい起きていたか」を、アウトカムと共変量の関係をモデル化して**前向きシミュレーション**によって求めます。

### 7-2. 2つの部品

* **部品 (A) アウトカムモデル（ハザードモデル）**: その区間でイベントが起きる確率を A・L・baseline共変量 で予測する。
* **部品 (B) L の遷移モデル（★g-formulaの心臓★）**: 次の区間の HbA1c 高値の確率を、前区間の治療 Aprev、前区間の Lprev、baseline共変量 で予測する。
  * 「増量すると次の月の HbA1c が下がる（A → L）」という時間依存交絡の動的な遷移をこのモデルが担います。**MSMには存在しない、g-formula特有のモデルです**。

### 7-3. 前向きシミュレーションの3ステップ

各時間区間（1から6）をループし、以下の3つを繰り返します。

1. **L のサンプリング**: (B)の遷移モデルから、その区間の L (HbA1c) を予測・サンプリングする。
2. **ハザード $h_k$ の計算**: サンプリングされた L と、固定した治療介入 A（全員 1、または全員 0）を (A)のアウトカムモデルに入れてハザードを求める。
3. **生存確率 $(1 - h_k)$ の掛け合わせ**: 生存確率を累積していく。

> **【つまずき手当て 3：なぜ掛け合わせるのか】**
> 「6区間すべてを生き延びてはじめて『最後まで無事』と言える。各区間を生き延びる確率は $(1-h_1) \times (1-h_2) \times \dots \times (1-h_6)$ という**AND条件**（掛け算）で求まる。その余事象（1から引いたもの）が累積イベント発生率になる」
> この生存確率の概念を、マスを埋めていく絵でイメージします。

> **【つまずき手当て 4：反実仮想の代入とは】**
> コード上の `A_k <- rep(1, N)` という一行が、「全員が増量状態にある反実仮想の世界」をデータ上で構築している操作そのものであることを直感的に捉えます。

### 7-4. 【最大の仕掛け：伏線の回収】

事前課題（`01`）で真の因果効果（`−0.232`）を算出したコードと、本番の g-formula のシミュレーションコードは**構造が全く同じ**です。

* 違いは、データ生成ルールである「真のパラメータ」をそのまま使って回したか、データから「推定したパラメータ」を使って回したかだけです。
* スクリプト内で、モデルの推定値の代わりに「真の係数」を直接代入してシミュレーションを行うと、**RD = −0.231** となり真値にほぼ一致します。これにより、「g-formula とは、データ生成過程（DGP）をデータから推定し、それを用いて介入シミュレーションを行う手法である」という本質が腑に落ちます。

### 7-5. g-formula 推定結果

* **g-formula リスク差 = −0.240**、95%CI `[−0.304, −0.168]`（Bootstrap値）

## 8. MSM と g-formula は何が違うのか

### 8-1. モデル化する対象の違い

| 手法                 | モデル化の対象                                 | 主な前提                                         |
| :------------------- | :--------------------------------------------- | :----------------------------------------------- |
| **MSM (IPTW)** | **治療の割り付けプロセス**               | 治療モデルの定式化が正しいこと                   |
| **g-formula**  | **アウトカムおよび共変量の遷移プロセス** | アウトカムモデル・遷移モデルの定式化が正しいこと |

* **なぜ両方やるべきなのか？**:
  実際の解析において、モデルが「本当に正しいか」は誰にも証明できません。しかし、**全く異なるルート（治療側モデル vs アウトカム側モデル）でアプローチした2つの手法が、同じデータから同様の推定値（−0.280 と −0.240）を叩き出したとき、どちらのモデルもそれなりに妥当に回っていると推測できます**。これが、ガイドライン等で両方の手法の実行が推奨される所以です ([00:57:08](https://www.youtube.com/watch?v=huS1yp6DtlI&t=3428s))。

### 8-2. 各推定値の比較まとめ

| 推定量               | リスク差 (RD)     | 特徴                                           |
| :------------------- | :---------------- | :--------------------------------------------- |
| **真値**       | **−0.232** | シミュレーション上の絶対的基準                 |
| naive (L非調整)      | −0.110           | 時間依存交絡により過小評価                     |
| naive (L調整)        | −0.093           | 中間変数バイアス（過剰調整）によりさらに壊れる |
| **MSM (IPTW)** | **−0.280** | 真値の少し上（bootstrap で不確実性を評価）     |
| **g-formula**  | **−0.240** | 真値の少し下（真のパラメータ代入時は −0.231） |

* **完全一致しない理由**: 小標本（N=1000）による推定のブレによるものです。そのため信頼区間に幅があり、不確実性が誠実に表現されています。

### 8-3. 【実務（Breakthrough Stroke研究）との違い】

臨床研究（Breakthrough Stroke研究）では治療戦略がベースラインで固定されているため、治療がその後の L を動かすプロセスを追う必要がなく、**L の遷移シミュレーション（ステップ1）が不要**になります。その結果、g-formula は単なる「ベースライン共変量による標準化（standardization）」に退化します。

## 9. 発展：時間依存交絡を3個に増やす（`10_advanced.R`）

実際の解析では、時間依存交絡が複数（例：HbA1c、腎機能低下、体重増加）存在するのが普通です。交絡因子を1個から3個に増やすと、2つの手法で**実装手間の非対称性**が浮き彫りになります。

* **MSM**: 重みの分母予測モデルに新しい変数（L2, L3 とその履歴）を足すだけであり、実装の手間はほとんど変わりません。
* **g-formula**: 増えた交絡因子の数だけ、個別の「遷移モデル」とサンプリング処理を記述する必要があり、手書きの実装難易度が急上昇します。
  * *解決策*: ここで実務用の **`gfoRmula` パッケージ**を使用します。
  * *メッセージ*: パッケージが中で行っているのは、`09_gformula.R` で手書きした前向きシミュレーションと**全く同じ**です。手書きで泥臭さを体感しているからこそ、パッケージがブラックボックスにならず、安心して使用できます。

## 10. まとめ：なぜ Breakthrough Stroke 研究では time-varying 解析を「やりたくてもできなかった」のか

本ハンズオンでは time-varying な交絡を扱い、MSM と g-formula で真値に迫ることに成功しました。しかし実際の臨床研究（Breakthrough Stroke研究）では、この解析を断念し、より単純なベースライン固定の ITT 解析に退きました。

理由は、time-varying 解析が要求する**データ要件の厳しさ**にあります。

1. **時間依存交絡 L が測定できていること**:
   毎区間の HbA1c や血圧などの検査推移が必要です。しかし、**レセプト（claims）データにはこれらの臨床検査値そのものが含まれていないことが多い**ため、遷移モデルや分母モデルを組むこと自体が不可能です。
2. **Positivity 違反の深刻化**:
   治療の軌跡を時間依存で細かく追うと、「その治療パターンをたどる患者」が実データ上で極端に少なくなり、重みが爆発するか、標準化が外挿（予測モデルの無理な引き延ばし）に依存することになります。

> 💡 **「手法が高級であるほど、それを支えるデータ要件も厳しくなる」**
> 前提条件を満たせない高級な手法を無理に使うよりも、前提を満たせる頑健な推定量（ベースライン固定の退化版）を採用することこそが、実務における誠実で正しい判断です。

なお、MSM と g-formula を学んだだけでは、まだ臨床研究は完成しません。現実には、脱落に対する IPCW、競合リスク、欠損値の処理など、本日は触れなかった多くのピースが必要となります。今回はその中心にある2大エンジンを、手を動かして理解した、という位置づけになります。

### 11-2. デザイン段階における「二の矢（代替案）」の準備

現実のデータベースはデータを実際にみないと分からないことが多々あります。

* **鉄則**: 最初に一番難しいデザイン（time-varying）で計画を立てておけば、データの制約（positivity違反など）が判明したときに、簡単なデザイン（ベースライン固定のITTなど）へ退くのは容易です。しかし、逆は不可能です。
* 解析計画を立てる段階で、**「もしデータ数が足りなかったり、positivityが崩れたりした場合は、ITT（ベースラインでの傾向スコア）に退く」という二の矢（代替案）**を必ず用意しておきましょう。
