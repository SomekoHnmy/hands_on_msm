# MSM・g-formula ハンズオン 解説・進行ガイド

**題材：メトホルミン単剤治療における「増量する／据え置く」という時間依存の治療判断が、心血管イベントに与える因果効果**

このドキュメントは、講師用の進行台本と、参加者用の読み物を兼ねています。ハンズオンの狙いは、MSM（周辺構造モデル）と parametric g-formula という2つの手法を、**疫学を少しかじった人が「手を動かして腑に落ちる」**ところまで持っていくことです。目新しい題材ではなく、因果推論の教科書で繰り返し使われてきた定番の構成を、**とにかく分かりやすく丁寧にやり切る**ことを目的にしています。

---

## 0. なぜこのハンズオンをやるのか

MSM や g-formula は、時間依存交絡（time-varying confounding）を扱うための標準的な手法です。しかし、疫学をかじった人でも「名前は知っているが何をしているのか分からない」ことが多い。その一因は、既存の解説が「分かっている人が、分かっている人向けに」書かれていて、初学者が最初につまずく段差が放置されがちなことにあります。

このハンズオンでは、その段差を一つずつ埋めます。具体的には、初学者が必ず引っかかる次の5点に、教材の各所で手当てをしています。

1. **なぜ「調整」ではダメなのか**（共変量に入れるのと重み付けは何が違うのか）
2. **重みの分子・分母が何の確率なのか**
3. **g-formula の「各期間のハザードを掛け合わせる」の意味**（なぜ足さず掛けるのか）
4. **g-formula の「反実仮想の代入」が何をしているのか**（`A <- 1` の一行の意味）
5. **MSM と g-formula は同じestimandを違うルートで推定していること**

## 2. 舞台設定：登場人物を具体物で固定する

抽象記号 `L`・`A`・`Y` だけで話を進めると、初学者はここで脱落します。だから**記号を必ず具体物とセットで**扱います。

- **A（治療）** = その月にメトホルミンの「増量状態にある（=1, 初期用量より高い用量で治療中）／初期用量のまま（=0）」
- **L（時間依存交絡）** = その月に HbA1c が「高値（=1）／良好（=0）」
- **Y（アウトカム）** = その月に心血管イベントが起きたか

時間は1か月刻み × 6区間。baseline共変量（時間で変化しない）として年齢・喫煙・高血圧・脂質異常症の4つ。

> **【治療を「状態」で捉える理由】** A を「その月に増量したというアクション」ではなく「その月に増量状態にあるか」という状態で定義します。こうすると、month=1 から増量状態の人・そうでない人の両群が存在し（アクションだと前月がない初月に群が作れない）、「常に増量状態でいく戦略 vs 初期用量を維持する戦略」という持続的な戦略の比較が自然に定義できます。据え置き・減量・中止は「増量状態でない（=0）」に含めます。

### この題材の「肝」＝ L が3つの顔を同時に持つこと

これが、MSM も g-formula も必要になる**唯一の状況**の核心です。板書やスライドで、矢印を一本ずつ指しながら説明してください。

- **① L → A**：HbA1c が高い → 増量状態に入りやすい（Lは治療判断を駆動する = 交絡）
- **② L → Y**：HbA1c が高い → 心血管イベントが多い（Lは予後を左右する = 交絡）
- **③ A → L**：増量状態を続ける → 次の月の HbA1c が下がる（Lは過去の治療の結果 = 中間変数）

①②だけなら L はただの交絡因子で、回帰に入れれば済みます。**問題は③**。③があると、L は「これから受ける治療の交絡因子」であると同時に「すでに受けた治療の中間変数」になります。

> **【つまずき手当て 1：なぜ調整ではダメか】**
> ここが第一の関門です。「L を回帰に入れれば交絡は取れるのでは？」という素朴な発想が、なぜ通用しないのかを、③の矢印を指して説明します。
> - L を**入れない** → ①②の交絡経路が開いたまま → 交絡バイアス
> - L を**入れる** → ③の「増量→HbA1c改善→イベント減」という**本来評価したい経路**まで塞ぐ（過剰調整・中間変数バイアス）
>
> つまり「入れても入れても壊れる」。この一点を、事前課題で参加者自身の手で確認させます（後述）。

## 3. 事前課題（`01_prework.R`）

### 3-1. 真値：仕組みを知っているからこそ計算できる「正解」

シミュレーションの最大の強みは、データの作り方（DGP）を自分で決めるので、**正しい答え（真値）を後から計算できる**ことです。大規模な集団に「全員かならず増量状態」「全員かならず初期用量のまま」を強制し、それぞれの6か月累積発生率を求め、その差を真の因果効果とします。

**真値：リスク差 = −0.233**（常に増量状態でいくと、心血管イベントを約23%ポイント減らす）

> 現実の観察データでは DGP を知らないので、この計算はできません。**だからこそ MSM や g-formula で「推定」する必要がある**、という動機づけをここで与えます。

### 3-2. 生データを person-interval に畳み込む

事前課題では、現実に近い**バラバラの生データ（4テーブル）**を配布します：患者マスタ（patients）、処方ログ（prescriptions、用量 mg）、検査ログ（labs、HbA1c 実値）、イベントログ（events）。これを、1人×月で1行の **person-interval 形式**に、`for` ループで一段ずつ畳み込みます（パッケージには頼りません）。ここで「その月の用量が初期用量 500mg より高いか」で増量状態（metformin_high）を導出し、HbA1c 実値を 7.5% で二値化します。本番のすべてがこの person-interval の上で動くので、ここで構造に馴染んでおきます。「生ログを月グリッドに畳む」のは実データ解析の出発点そのものです。

### 3-3. naive 解析：「調整してもしなくても真値に届かない」

ふつうの回帰（pooled logistic regression）を2通り試します。

| 推定量 | リスク差 |
|---|---|
| **真値** | **−0.233** |
| naive（L を調整しない） | −0.110 |
| naive（L を調整する） | −0.094 |

**メッセージはシンプルに**：L を入れても入れなくても、真値 −0.23 には届かない。調整の仕方で値は動くが、どちらが正しいとも言えない。大事なのは「どちらの調整が正しいか」ではなく、**ふつうの回帰という枠組みそのものが、この問題には力不足**だということ。

> ここで各バイアスの機序（交絡か過剰調整か）を厳密にラベル付けしようとすると、小標本では数値が毎回きれいには対応せず、かえって混乱します。「どちらも真値に届かない」という事実だけで十分です。

事前課題はここで終わり、「**では、どうすればよいのか？**」という問いを残して本番へ引き継ぎます。

## 4. 本番① MSM（`02_msm.R`）：治療側からモデル化する

### 4-1. 考え方（一言で）

「HbA1c が高い人ほど増量状態に入りやすい」という**治療の偏り**を、逆確率で重み付けして打ち消し、あたかもランダムに治療が割り振られたかのような**疑似集団**を作る。その疑似集団で増量状態 vs 非増量を比べれば、交絡の影響が取り除かれる。

### 4-2. 安定化重みの2つのモデル

- **分母モデル**：交絡を**すべて**入れて治療を予測する（時間依存交絡 L も、履歴 Lprev/Aprev も入れる）。治療の偏りを正確に捉える。
- **分子モデル**：baseline と治療履歴だけで予測する（L は入れない）。重みを安定させるための調整。

安定化重み = （分子の確率）/（分母の確率）を、各人で時間方向に掛けたもの（累積積）。

> **【つまずき手当て 2：重みの分子・分母が何の確率か】**
> 式を見せる前に、**具体的な1人の患者（id=3）で数値を1周させます**（スクリプトが患者id=3の重みの手計算テーブルを出力します）。「この人は HbA1c 高値なのに、確率 fd でこの治療状態にあった。fd が小さい＝“受けにくい治療をあえて受けた人”ほど、重みが大きくなって疑似集団で強く効く」と、テーブルの行を指しながら説明します。式は数値の後に回す。

### 4-3. 実務チェック（第2部）―― 既習者向けの見どころ

参加者には MSM/g-formula を既に学んだ人も含まれます。そこで `02_msm.R` は第1部（基本）に続けて**第2部（実務チェック）**を置き、実データで MSM を回すときの確認項目を一通り再現します。ここが既習者の持ち帰りどころです。項目は、①モデル収束、②positivity（各月のESS）、③重み分布と truncation、④各時点のバランス（SMD）、⑤クラスターロバスト SE、⑥bootstrap CI。実務では cobalt・sandwich 等のパッケージを使いますが、教材では依存を減らすため base R で自前計算します。

**positivity は各月の有効サンプルサイズ（ESS）で確認する。** 重み付けで実効的な情報量がどれだけ残るかを各月で見ます。このデータでは各月 ESS が **88〜96%** と高く、重みの最大値も 2 前後と穏やか。どの月・どの交絡水準でも両群が十分存在し、極端な重みもない＝**positivity は良好**です。

**月次 SMD は、このシミュレーションデータでは 0.1 以内に収まりません（正直に見せる）。** 重み付け後も各月の SMD、特に hba1c_high が 0.2〜0.5 と目安 0.1 を超えます。ここは取り繕わず、「SMD は月次で見る → このデータでは破綻するが、今回は MSM の一連の**手順**を見てもらうのが目的なので、このまま進む（positivity は ESS で確認済み）」と淡々と扱います。

> **【なぜ月次 SMD が締まらないか（深追いはしないが、問われたら）】** hba1c_high は交絡因子であると同時に**前月治療の結果（中間変数）**でもあります。増量状態の人は増量の効果で HbA1c が下がっているので、各月断面で「増量群は HbA1c 低め」という差が出ます。この差は交絡の残りではなく**治療効果の反映**を含むため、各月の素朴な SMD は締まりにくく、完全に締めると治療効果まで消してしまう。だから ESS（positivity）で健全性を確認しつつ、バランスは全期間の推定結果（真値への近さ）で判断する、という立て付けにしています。崩れているのは主に hba1c_high と連続変数の age_z で、二値の baseline 共変量（高血圧など）はおおむね 0.1 未満に揃います。

### 4-4. 結果

**MSM リスク差 = −0.225、95%CI [−0.296, −0.136]**（bootstrap）

naive では −0.09〜−0.11 で頭打ちだったのが、同じデータから MSM では真値 −0.233 に迫れました。

> 累積曝露 `cumA`（これまで増量状態にあった月数）を曝露に入れている点に注意。増量の効果は1区間だけでなく、続けることで積み上がる（A→HbA1c改善→イベント減が毎区間積み重なる）ため、「6か月ずっと増量状態でいた戦略」の効果を正しく表現するのに必要です。

### 4-5. 【この MSM は論文と何が違うか】（末尾3分）

ここで作った治療重みは**時間依存**でした：各区間で L（HbA1c）を見て、`P(A_k | Lprev, Aprev, L_k, baseline)` を分母に、履歴を掛け合わせています。

一方、**breakthrough-stroke 論文**では治療戦略が baseline で一度決まる（once-treated-always-treated, ITT推定量）ため、この時間依存の治療重みは baseline の傾向スコア1回分に**つぶれます**。時間依存で残るのは、脱落（disenrollment）に対する打ち切り重み（IPCW）だけ。つまり論文の MSM は、今日の手続きの**特殊ケース（退化版）**にあたります。なお論文が各月 SMD を 12/12 balanced にできたのは、治療が baseline 固定で feedback による中間変数化が起きないから。時間依存治療で feedback があると各月バランスは素朴には締まらない、というのは今日のデータで見た通りです。

## 5. 本番② parametric g-formula（`03_gformula.R`）：アウトカム側からモデル化する

### 5-1. 考え方（一言で）

「もし全員が毎区間かならず増量状態でいたら／初期用量のままだったら、心血管イベントは何％起きるか」を、アウトカムと共変量のモデルを使って**前向きにシミュレーション**して求める。

### 5-2. 2つの部品

- **(A) アウトカムモデル（ハザードモデル）**：この区間でイベントが起きる確率を A・L・baseline で予測。
- **(B) L の遷移モデル**★g-formulaの心臓★：次の区間の HbA1c 高値を、前区間の治療 Aprev・前区間の L・baseline で予測。「増量すると次の HbA1c が下がる（A→L）」を、このモデルが担う。**MSMにはなかった部品**。

### 5-3. 前向きシミュレーション本体：3ステップ

各区間でやることは、たった3つ。これを区間1から6まで繰り返します。

1. **この区間の L をサンプリング**（L遷移モデルから）
2. **この区間のハザード h_k を計算**（その L と、固定した治療 A で、アウトカムモデルから）
3. **生存確率 (1 − h_k) を掛けていく**

> **【つまずき手当て 3：なぜ掛け合わせるのか】**
> 第三の関門にして、この教材で最も時間をかけるべき所です。「6区間すべてを生き延びてはじめて“最後まで無事”。各区間を生き延びる確率 (1−h_1)、(1−h_2)、… を**全部掛けたもの**が『6か月間ずっと無事でいる確率』。その余事象（1から引いたもの）が累積発生率」――と、6個のマスを一つずつ埋めていく絵で説明します。「なぜ足さず掛けるのか＝“毎区間、連続して生き延びる”という**AND条件**だから」を強調。

> **【つまずき手当て 4：反実仮想の代入とは】**
> 「治療を固定する」のはコードでは `A_k <- rep(1, N)` の一行ですが、これが「全員が増量状態にある反実仮想の世界を作っている」という意味だと直感させます。後述の伏線回収（§5-4）が、ここを決定的に補強します。

### 5-4. 【伏線の回収】g-formula は「真値を出したコード」とそっくり

これがこのハンズオン最大の仕掛けです。

事前課題で真値を出したとき、**真の係数**を使って介入世界をシミュレーションしました。本番の g-formula がやっているのは、**まったく同じこと**。違いは「真の係数」を使うか、「データから推定した係数」を使うかだけ。

スクリプトでは、真の係数をそのまま前向きシミュレーションに入れてみせます。すると **RD = −0.231** と真値（−0.233）にぴたりと一致します。

> **これで参加者は「g-formula とは、DGP をデータから推定して、それを介入下で回す手法だ」と視覚的に理解します。** 同じループを2回見せて、違うのはパラメータの出所（真の値か、推定値か）だけ、と示すのがポイント。反実仮想の代入（つまずき4）と掛け合わせ（つまずき3）が、ここで一つの絵に統合されます。

### 5-5. 結果

**g-formula リスク差 = −0.221、95%CI [−0.281, −0.162]**（bootstrap）

### 5-6. MSM と g-formula を並べる

| 推定量 | リスク差 |
|---|---|
| 真値 | −0.233 |
| naive（L 非調整） | −0.110 |
| naive（L 調整） | −0.094 |
| MSM（IPTW） | −0.225（真値の少し下） |
| g-formula | −0.221（真値の少し下） |

> **【つまずき手当て 5：2つは同じものを違うルートで推定している】**
> naive は真値から大きく外れるのに対し、MSM と g-formula は**まったく別のルート**（治療側 vs アウトカム側）を通りながら、どちらも真値 −0.23 の近傍を指しています。
>
> **完全一致しないことは、むしろ正直で良い**。もしぴったり一致したら「どちらか一方でいい」「何かのトリックでは」と誤解を招きます。近い値で同じ方向を向いていることこそ、独立な2経路が同じ結論を支持している証拠。完全一致しないのは N=1000 の小標本で推定がぶれるためで、だからこそ bootstrap の CI は広めに出ます。それが正直な不確実性の表現です。

> **【seed について、賢い参加者への備え】**
> 乱数シードは 123 に固定しています（再現性のため）。「都合のいいseedを選んだのでは？」と突っ込まれたら、こう答えます――その通り、これは典型的な引きを見せるために選んだ1つのデータセット。DGP（真の仕組み）はいじっていない。現実の研究者も手元のコホートは1つで、それが良い引きか悪い引きかは選べない。同じDGPでも別のデータでは値がぶれる。それこそが小標本のリアルで、bootstrap の CI の広さがそのぶれを表している。突っ込みを教育の好機に変えられます。

### 5-7. 【この g-formula は論文と何が違うか】（末尾3分）

今日の g-formula の心臓は「L の遷移モデル（ステップ1）」でした。増量状態が次の HbA1c を変える（A→L）フィードバックを、このモデルが前向きに再現します。

論文では治療が baseline で固定（once-treated-always-treated）なので、「治療がその後の L を動かす」を追う必要がない。すると **L の遷移シミュレーション（ステップ1）が丸ごと不要**になり、g-formula は「baseline共変量での標準化」へと退化します。論文の g-formula は、今日の time-varying 版の**特殊ケース**です。

## 6. 発展：時間依存交絡を3個に増やす（`04_advanced.R`）

現実の解析では時間依存交絡は複数あるのがふつうです（HAART の古典研究でも、実際には CD4 だけでなく HIV ウイルス量など複数を同時に扱っていました。教科書が CD4 一つで説明するのは分かりやすさのための単純化であって、手法の制約ではありません）。

交絡を1個から3個（HbA1c・腎機能低下・体重増加）に増やすと、MSM と g-formula で**手間のかかり方がまったく違う**ことが見えます。

- **MSM**：重みの分母モデルに項（L2, L3 とその履歴）を足すだけ。手続きはほぼ変わらない。
- **g-formula**：交絡ごとに遷移モデルと前向きサンプリングが要る。手書きだと急に重くなる。→ だから **gfoRmula パッケージ**に任せる。

> ここで gfoRmula パッケージを初めて登場させます。大事なのは「パッケージが中でやっているのは、`03_gformula.R` であなたが手で書いた前向きシミュレーションと**まったく同じ**」と種明かしすること。1交絡版で中身を手書きして理解しているから、パッケージはブラックボックスにならない。「手書きで重さを体感 → パッケージがそれを引き受ける」という物語で締めます。
>
> この「手間の非対称性」自体が、実務でどちらの手法を選ぶかの判断材料の一つになります（正解は状況次第で、両方できるのが理想）。

## 7. まとめ：なぜ論文は time-varying 解析を「やりたくてもできなかった」のか（締め・約5分）

このハンズオンの本当の締めくくりであり、最も実務的な教訓です。

今日、私たちは **time-varying な交絡（毎区間 HbA1c が動き、治療状態を左右し、治療で動く）** を扱い、MSM と g-formula で真値に迫りました。しかし breakthrough-stroke 論文では、これをやりたくても**断念**し、より単純な **ITT（baseline で治療戦略が固定、once-treated-always-treated）** に退きました。なぜか。

理由は、time-varying の MSM / g-formula が成り立つ**大前提**にあります。

1. **time-varying confounder L が測れていること。** 今日の題材では HbA1c という L が毎区間観測できる前提でした。しかし**レセプト（claims）データには、HbA1c や血圧のような検査値そのものが入っていないことが多い**。脳卒中の重症度に相当する time-varying な L が測れなければ、L→A のフィードバックをモデル化できず（＝MSMの分母モデルも、g-formulaのL遷移モデルも組めない）、time-varying 解析は成立しません。

2. **time-varying にすると positivity 違反が深刻化する。** 特定の治療軌跡（例：ずっと増量し続ける／ずっと据え置き続ける）をたどる人が実データではほとんどいなくなり、重みが極端になったり、標準化が外挿に頼ったりします。

だから論文は、**測れない L に依存しない、より頑健な ITT（baseline固定）の推定量**を選びました。これは「手を抜いた」のではなく、**データ要件を満たせない高級な手法に無理に手を出すより、前提を満たせる推定量を選ぶ」という、正しい判断**です。

### 一言でいえば

> **手法が高級であるほど、それを支えるデータ要件も厳しくなる。**
> time-varying の MSM / g-formula は強力だが、「毎区間、交絡 L が測れている」という前提の上でしか成り立たない。その前提が満たせないなら、baseline固定の推定量（今日見た手法の“退化版”）に退くのが誠実。今日のハンズオンで扱った「理想の解析」と、論文で実際に採った「現実的な妥協」の差は、まさにこのデータ要件の差から生まれています。

そして最後に、正直に付け加えるべきこと――**MSM と g-formula を学んだだけでは、まだ研究はできません。** 現実には、脱落に対する IPCW、positivity の診断、モデルの誤設定への感度分析など、今日は触れなかった多くのピースが必要です。今日はその中心にある2つのエンジンを、手を動かして理解した、という位置づけです。
