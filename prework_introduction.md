# 因果推論への招待 ―― HbA1c 7.3% の患者を前に

**MSM・g-formula ハンズオン 事前読み物（第0場〜第6場）**

> この読み物は、当日のハンズオンに入る前に、「そもそも因果推論とは何か」を、一人の患者をめぐる問いから考えていくものです。数式もコードも出てきません。読み終えると、当日扱う marginal structural model (MSM) と g-formula が「なぜ必要になるのか」が腑に落ちているはずです。あわせて事前課題スクリプト（`02_prework_pooled_logistic_regression.R`）に触れておくと、当日の理解がぐっと深まります。ただ、読まずに当日を迎えても大丈夫なように作ってあります。

## 第0場：ある外来

![ある外来の様子](illustrations/scene_0.jpeg)

65歳の男性が、あなたの外来を受診しています。糖尿病でメトホルミンを飲み続けて数年。今日の HbA1c は 7.5%で、少し高めです。ガイドライン的には、もう一段の血糖コントロールを目指して増量を考えてもいい数字です。

若い患者なら、あなたは迷わないと思います。7.5% なら増量、あるいは薬を足します。標準的な判断です。

でも、この患者は65歳です。腎機能は年齢なりに落ちています。一人暮らしで、低血糖を起こしたときに気づいてくれる人はいません。そして診察の終わりに、こう言うのです。

> 「先生、薬、このままでいいかね。……それとも増やした方がいいのかい。増やしたところで、どれくらい違うもんなんかね。低血糖が怖い歳で、たいして変わらんのなら、これ以上薬は増やしたくないよ」

さて、あなたはどうしますか。

「まあ、この歳だし、7.5% くらいなら、いいか」と据え置くでしょうか。「いや、ここで低血糖を恐れずにメトホルミンを増量してHbA1cを下げておいた方が、将来の心筋梗塞や脳卒中を防げるはずだ」と増量に踏み切るでしょうか。

これは、悩みます。若い患者では悩まないこの判断が、この患者では本当に悩ましい。そして、その悩みの正体を突き詰めていくと、実は「因果推論」という考え方の入り口に、あなたは立っています。

この読み物は、この患者をめぐる一つの問い ――

**「メトホルミンを増量することは、この患者の将来の心血管イベントを、（どれくらい）減らすのか？」**

―― を、最後まで手放さずに進んでいきます。

## 第1場：因果推論とは、特別な魔法ではない

![因果の問い](illustrations/scene_1.jpeg)

「因果推論」と聞くと、何か難しい統計手法の名前のように思うかもしれません。当日扱う MSM や g-formula も、その一種です。でも、まず誤解を解いておきたいことがあります。

**因果推論とは、「原因と結果の関係を知りたい」という問いそのもの**のことです。特別な手法の名前ではありません。

あなたが今抱いている「増量は心血管イベントを（どれくらい）減らすのか」という問いは、まぎれもなく因果の問いです。「増量」という原因が、「心血管イベントが減る」という結果を引き起こすのか。これを問うている時点で、あなたはもう因果推論をしています。

ひとつ、小さいけれど大事なことを添えておきます。臨床の現場では、私たちはつい「効くのか、効かないのか」と、白黒で問いを立てがちです。でも、さきほどの患者が求めていたのは、それではありませんでした。「増やしたところで、どれくらい違うのか」。彼が知りたいのは、効くか効かないかではなく、**どれくらい**効くのか ―― その"量"です。低血糖のリスクを負ってまでやる価値があるかは、この量で決まります。因果推論が本当に答えるべきなのは、多くの場合この「どれくらい」の方 ―― 効果の大きさ（効果量）です。この読み物の問いを「（どれくらい）減らすのか」とカッコ書きにしたのは、そのためです。

だから、因果の問いに答えようとする研究は、実にさまざまです。横断研究で「増量している人としていない人で、心血管イベントの有無に差があるか」を見ることもできます。これも、広い意味では因果の問いに迫ろうとする一つの試みです。

ただ、方法によって、**真実にどれだけ近づけるかが異なります**。横断研究は手軽ですが、まぎれもの（バイアス）が多く混じり、真実からは遠くなりがちです。もっと丁寧に、まぎれものを一つずつ取り除いて、真実に近づこうとする方法があります。

**MSM や g-formula は、その「まぎれものをより多く取り除いて、因果の真実に近づく」ための道具**です。因果推論そのものではなく、因果推論を精度よく行うための道具、という位置づけです。この区別を、まず頭の隅に置いてください。「因果推論＝MSM/g-formula」ではありません。因果の問いは誰でも立てられます。問題は、それにどれだけ正確に答えられるか、なのです。

## 第2場：反実仮想 ―― 見ることのできないもう一つの世界

![反実仮想のイメージ](illustrations/scene_2.jpeg)

では、あなたの問いに「正確に」答えるとは、どういうことでしょうか。

「増量が心血管イベントを減らす」と本当に言い切るには、理想的には、こうしたいところです。

**この患者に増量した世界で、彼が心筋梗塞を起こすかを見ます。そして、まったく同じこの患者に、増量しなかった世界で、彼が心筋梗塞を起こすかを見ます。その二つの世界を比べます。**

もし「増量した世界」では起こさず、「増量しなかった世界」では起こしたなら ―― そのとき初めて、「増量が、この患者の心筋梗塞を防いだ」と、胸を張って言えます。二つの世界の違いは増量したかどうかだけで、患者も、年齢も、腎機能も、何もかも同じなのですから。差が出たなら、それは増量のせいだと断定できます。

この「増量しなかった、もう一つの世界」のことを、**反実仮想（counterfactual）** と呼びます。事実に反する、仮想の世界。因果を語るとは、つまるところ、この反実仮想と現実を比べることなのです。

ところが ―― ここに、どうしようもない壁があります。

**私たちは、同じ患者について、増量した世界と増量しなかった世界を、両方見ることはできません。** 実際に増量すれば、増量しなかった世界は永遠に見られません。据え置けば、増量した世界は見られません。人生は一度きりで、片方の世界しか実現しません。

これを、**因果推論の根本問題**と呼びます。因果を知りたいのに、そのために本当に必要な「両方の世界」は、原理的に手に入りません。

だから、私たちがやっていることは、突き詰めれば一つです。**見ることのできない反実仮想を、どうにかして"それらしく作ること"です。** その作り方の巧拙が、因果推論の精度を決めます。

## 第3場：RCT ―― 純粋な答え、しかし限界

反実仮想を作る方法として、最も純粋で強力なのが、**ランダム化比較試験（RCT）** です。

同じ患者で両方の世界を見られないなら、せめて **「増量した患者たち」と「据え置いた患者たち」を、"そっくりな二つの集団"にします**。年齢も、腎機能も、喫煙も、あらゆる背景がそっくりな二つの集団を作り、片方だけ増量します。両者にきれいに差が出れば、それは増量のせいだと言えます。片方の集団が、もう片方の「反実仮想」の役を務めるわけです。

どうやってそっくりにするのでしょうか。**くじ引き（ランダム化）** です。増量するか据え置くかをコイン投げで決めれば、背景のあらゆる要素 ―― 測れるものも、測れないものも ―― が、平均的に二つの集団で釣り合う。これがランダム化の魔法です。

もし、あなたの問いに純粋に答えたいなら、答えは明快です。**RCT をやれば良いのです。**

……でも、現実にはそうはいきません。

世の中のあらゆる臨床疑問に RCT をやることはできません。費用も時間もかかりすぎます。そして今回の問いには、さらに厄介な事情があります。あなたが本当に知りたいのは、単に「増量 vs 据え置き」という一回きりの判断ではないはずです。**「これから半年、毎月 HbA1c を見ながら、高用量にする方針でいくか、多少高めでも用量を抑え気味で様子を見る方針でいくか」** という、**時間をかけて続けていく"方針（戦略）"の比較**です。

こういう「経過を見ながら調整し続ける戦略」を RCT で検証するのは、単発の薬 A vs 薬 B を比べる RCT よりも、はるかに設計が難しいのです。だからこそ、すでに手元にある観察データ ―― 日々の診療で蓄積された、実際に増量した人・据え置いた人のデータ ―― から、なんとか答えに迫りたくなるのです。

## 第4場：観察研究で反実仮想に近づく ―― 二つの発想

RCT が使えないなら、観察データで反実仮想（あらゆる背景がそっくりな二つの集団）を"作り"にいきます。その発想には、大きく二つの系統があります。当日扱う MSM と g-formula は、それぞれこの二つに対応します。ここは後で効いてくるので、頭に入れておいてください。
なお、このようにあらゆる背景がそっくりで、入れ替えてもかわりがないことをexchangeability（交換可能性）と言います。

**系統その1：治療を受ける確率で、集団の偏りを補正する。**

観察データでは、治療はくじ引きでは決まっていません。HbA1c が高い人ほど増量されやすい、というような偏りがあります。ならば、「この背景の人が、実際にこの治療を受けた確率」を計算し、その確率を使って集団の偏りを補正してやります。受けにくい治療をあえて受けた人を重く数え、受けやすい治療を受けた人を軽く数えます。こうして、あたかもくじ引きで治療が決まったかのような"疑似集団"を作ります。 ―― これが、当日の **MSM（重み付け）** につながります。

**系統その2：背景を揃えた予測から、反実仮想の世界を組み立てる。**

もう一つは、アウトカム（心血管イベント）そのものをモデル化する発想です。「この背景の人が、増量したらイベントは何％か」「据え置いたら何％か」を予測できるモデルを作り、それを全員に当てはめて、「もし全員が増量したら」「もし全員が据え置いたら」という反実仮想の世界を、丸ごと組み立てます。 ―― これが、当日の **g-formula（標準化）** につながります。

治療の側からアプローチするか（系統1）、アウトカムの側からアプローチするか（系統2）。**同じ反実仮想を、違う入り口から作りにいく二つの道があります**、と覚えておいてください。当日、この二つの道が、それぞれ MSM と g-formula という具体的な姿で再登場します。

## 第5場：時間が、すべてを難しくする

さて、ここまでは「増量した人 vs 据え置いた人」を、一枚の写真のように比べる話でした。ところが、実際のデータを開くと、そんな単純な話ではないことに気づきます。

一人の患者を、月ごとに追ってみましょう。1月は HbA1c が高くて増量しました。すると3月には HbA1c が下がりました。4月は落ち着いていたので初期用量に戻しました。すると6月にはまた上がってきて、また増量しました ―― 。

**治療は、一度決めたら終わりではありません。毎月、その時の状態を見ながら、増やしたり初期用量に戻したりを繰り返しています。** そして HbA1c の値も、それにつれて上がったり下がったりし続けています。

ここで、第4場の発想を素朴に当てはめようとすると、つまずきます。「高用量でいる人 vs 初期用量のままの人」を最初の時点で二群に分けて背景をそろえても、その後 HbA1c がどんどん変わり、それにつれて治療状態も動いていくので、最初にそろえた前提が時間とともに崩れていきます。ある月に背景をそろえても、次の月には HbA1c が動き、治療状態が変わり、また新たな偏りが生まれます。一枚の写真では捉えきれないのです。

ここが、この読み物で一番大事なところです。**なぜ、ふつうの解析ではダメなのか。**

もし、HbA1c のような「時間とともに変わる要素」が、治療とは無関係にただ揺れているだけなら、まだ何とかなります（通常の共変量として回帰モデルに含めて直接調整する通常の pooled logistic regression という方法で対処できます。具体的な方法は事前課題を参照してください）。
しかし、ダメなのは「時間依存交絡がある状況で、その交絡を通常の共変量として回帰モデルに入れて直接調整しようとすること」です。後述する MSM では、逆確率重み（IPW）を掛けた上で pooled logistic regression を推定することで、この問題を正しく解決します。つまり、pooled logistic という手法自体が悪いのではなく、重み付けをせずに直接調整しようとする使い方が力不足なのです。

問題は、HbA1c が次の三つの顔を**同時に**持っていることです。

- **HbA1c が高い → 増量したくなる**（HbA1c は、治療を決める理由になっている）
- **HbA1c が高い → 将来の心血管イベントが増える**（HbA1c は、予後を左右する）
- **増量する → 次の月の HbA1c が下がる**（HbA1c は、前の治療の結果でもある）

最初の二つだけなら、HbA1c はただの交絡因子です。解析で調整（考慮）してやれば済みます。ところが三つ目があるために、話が壊れます。

HbA1c を解析で**調整しないと**、「HbA1c が高い人ほど増量され、かつイベントも多い」という偏りが残り、増量がまるで効いていないように見えてしまいます。

かといって HbA1c を**調整すると**、今度は「増量した → HbA1c が下がった → だからイベントが減った」という、**まさに私たちが知りたい増量の効果の道筋**まで、消してしまいます。HbA1c は増量の"結果"でもあるので、それを揃えてしまうと、増量の恩恵の一部をなかったことにしてしまうのです。

**調整してもダメ、調整しなくてもダメ。** これが、治療で動く時間依存の交絡がもたらす、やっかいな罠です。ふつうの回帰という道具は、この罠の前で無力になります。

（この「調整してもしなくても真実に届かない」ことは、事前課題スクリプト `02_prework_pooled_logistic_regression.R` で、実際の数字として確かめられます。余裕があったら走らせてみてください。）

なお、今回のシナリオでは、「HbA1cが高くても、臨床家が薬剤を増量するか据え置くか、患者の背景を考慮してどちらの選択肢も選び得る」という前提で進めています。もし、「HbA1cが高い患者は、どの医師であっても100%必ず増量し、据え置くことは絶対にあり得ない」という極端な状況であれば、比較対照となる据え置き群がそもそも存在しなくなるため、解析自体が成り立たなくなります。

このように、個々の患者の背景（HbA1cなど）がどうであれ、いずれの治療戦略も選ばれる可能性がゼロではない（どの群にもある程度患者が存在する）状態のことを「positivity（正値性）」と言います。これは「exchangeability（交換可能性）」と並んで、MSMやg-formulaを成立させるために不可欠な大前提条件の一つです。
あともう１つ、consistency（臨床疑問で想定している介入と、データで実際に行われている介入がほぼ同じとみなせること）もありますが、今回はそれほど深入りしません。

## 第6場：そこで、MSM と g-formula

ふつうの解析が無力になるこの場面で、ようやく当日の主役が登場します。

発想はこうです。HbA1c を「その場で固定する共変量」として調整しようとするから壊れる。ならば、その扱いをやめればいい。HbA1c を固定するのではなく、治療との関係を別の形で処理する。

やり方は2つあります。ひとつは、HbA1c を **治療の受けやすさの予測に使い、その確率で重み付けする** (アウトカムのモデルには入れない)。もうひとつは、HbA1c を **治療に応じて動く変数としてシミュレートし、反実仮想の世界を組み立てる** 。

どちらも、1か月ごとに区切って各時点のモデルを作り、掛け合わせていくことで、「半年間ずっと高用量でいた世界」と「半年間ずっと初期用量のままだった世界」を組み立て、比べます。

この「各時点で区切ってモデルを作り、掛け合わせる」を、

- **治療の側から**（第4場の系統1）やるのが **MSM**
- **アウトカムの側から**（第4場の系統2）やるのが **g-formula**

です。第4場で分かれた二つの道が、HbA1c を固定するのではなく、治療との関係を別の形で処理するという工夫を経て、ここで MSM と g-formula という具体的な手法になって戻ってきました。

当日のハンズオンでは、この二つを実際に R で動かします。人工的に作ったデータ（正解＝本当のリスク差が分かっているデータ）を使い、ふつうの解析では正解に届かないこと、そして MSM と g-formula ではその正解に迫れることを、自分の目で確かめます。二つの手法が、違う道を通りながら、正解に近い答えにたどり着くのを見ることになります。

## 事前課題として、（可能であれば）やっておいてほしいこと

2つの事前課題スクリプト（[01_prework_generate_person_data.R](scripts/01_prework_generate_person_data.R) と [02_prework_pooled_logistic_regression.R](scripts/02_prework_pooled_logistic_regression.R)）で、次の二つに触れておいてください（できる範囲で構いません）。当日の理解が早くなると思います（必須ではありません）。

1. **person-time（person-interval）データを作る** ―― 一人の患者を「1か月ごとに1行」の形に並べ替える作業です。当日のすべての解析は、この形のデータの上で動きます。スクリプト（`01_`）で、その変換を一段ずつ手で追えるようにしてあります。
2. **pooled logistic regression解析で推定値を出す** ―― HbA1c を調整する場合としない場合の両方で、「1か月ごとに1行」の形に並べ替えたデータで増量の効果を推定してみます。スクリプト（`02_`）を実行し、どちらの推定値も、正解（真値）には届かないことを、数字で確認します。第5場で理屈として説明した「調整してもしなくてもダメ」が、実際の数字として立ち現れる瞬間です。

ここまでが事前課題です。**「では、どうすればいいのか」への答え ―― MSM と g-formula ―― は、当日、一緒に手を動かしながら解き明かしていきます。**

あの65歳の患者の問い、「増量は、彼の心血管イベントを本当に減らすのか」。その答えに、私たちがどこまで誠実に迫れるののでしょうか。それでは、当日お会いしましょう（もちろん後日録画された動画を見ながら学習することも可能です）。
